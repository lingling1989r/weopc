# 二、核心产品逻辑

核心逻辑：**以嘉宾优质灵活用工/副业项目为种子，以商业搜索为入口，串联「灵活用工供需对接+大众点评式信任背书+链接中转工具辅助+博客广场内容呈现」四大核心模块，采用前后端分离架构，MVP 1.0优先落地小程序快速冷启动，预留网页端拓展空间，前期靠免费功能积累精准流量（找钱方+招人方），后期复刻BOSS直聘「招人方付费提权+增值服务」的变现模式，同时渗透「先做牛马后做爷、稳健创业、透明副业」的价值观，坚守「所有灵活用工项目透明化」的宗旨，形成「工具引流→项目对接→点评信任→付费变现→内容沉淀（博客广场）」的闭环**，实现平台、嘉宾、招人方、找钱方四方共赢，兼顾当下冷启动效率与长期多端拓展，避免示范项目误解。

简化逻辑链：普通用户（找钱方）→ 小程序搜索首页找透明副业/灵活用工项目（查看项目细节、招人方优势、收益方式）→ 查看项目详情（报价/收益、执行要求、招人方优势、点评）→ 留资/对接项目提供方（招人方）→ 承接项目后发布点评→ 项目提供方（招人方）→ 小程序/运营后台（后续网页端）入驻，发布灵活用工/副业项目，清晰展示自身优势、项目细节、收益结算方式→ 免费获取精准找钱方客资→ 付费提权放大项目曝光→ 工具箱（链接中转/域名工具）→ 所有角色免费使用引流→ 联动项目页实现流量转化→ 增值服务变现→ 后台管理员→ 小程序/网页端后台审核灵活用工/副业项目（杜绝隐晦、违规）、管理客资、维护平台秩序、审核博客广场副业内容→ 博客广场→ 呈现优质灵活用工/副业项目案例、执行技巧、收益分享，强化「支持灵活用工、支持副业」的定位→ 后期网页端上线，同步所有核心功能及三角色权限，适配PC端高效操作场景（如招人方批量发布项目、管理员批量审核）。

## 2.1 前后端分离架构核心要求（适配多端+三角色）

为保障后续网页端顺利拓展及三角色权限管控，MVP 1.0开发阶段需遵循以下架构要求，确保多端复用、高效迭代、权限清晰：1.  后端：采用标准化接口设计，提供统一的数据服务，接口兼容小程序、网页端，同时区分三角色接口权限，避免越权操作，避免后续网页端开发重复开发后端逻辑；重点支持「灵活用工/副业项目」的发布、审核、展示接口，确保项目示例相关数据同步；2.  前端：小程序与后续网页端共用一套核心业务逻辑，差异化适配各端交互体验（如小程序适配移动端单手操作、网页端适配PC端多窗口操作），同时针对三角色设计专属操作界面（普通用户侧重找副业项目、项目提供方侧重发布/管理灵活用工项目、后台管理员侧重审核项目/内容），保持视觉调性、核心功能一致；3.  数据：用户数据、灵活用工/副业项目数据、客资数据、点评数据、工具使用数据、博客广场内容数据等，统一存储、实时同步，按角色权限分配数据查看/操作权限，确保小程序与网页端用户切换使用时，体验无缝衔接，权限同步；4.  运营后台：MVP 1.0可采用简易网页版（适配前期1-2人后台管理员运营），重点支持灵活用工/副业项目审核、博客广场内容审核功能，后续基于同一后端，优化网页端运营后台，强化三角色权限管控，实现更高效的批量操作、数据统计功能。

# 三、核心模块及功能需求（MVP 1.0，按落地优先级排序）

## 模块1：搜索首页（核心入口，落地优先级：最高）

### 3.1 模块定位

平台首页唯一核心，聚焦「搜索找透明副业/灵活用工项目」，无冗余内容，直击普通用户（找钱方）“找靠谱赚钱项目”的核心痛点，同时突出嘉宾项目、优质灵活用工项目曝光，适配前期「项目少、精准优先」的冷启动场景；基于前后端分离架构，适配小程序交互规范，预留网页端首页拓展接口，确保后续网页端首页功能同步、体验统一，同时区分三角色首页展示（仅普通用户可见完整首页，其他两角色跳转专属页面）；首页搜索提示、项目展示需贴合核心项目示例，避免误解。

### 3.2 核心功能需求

|功能名称|功能描述|交互细节|优先级|
|---|---|---|---|

### 🔧 技术架构决策

| 决策项 | 选择 | 原因 |
|--------|------|------|
| 项目所有权 | 一个 Provider 一个项目 | 简化管理，避免店铺概念复杂性 |
| 邀请码有效期 | 30 天 | 平衡安全性和易用性 |
| 短码长度 | 6 位 | 足够的唯一性，易于分享 |
| 积分系统 | 基于行为的奖励 | 激励 Provider 活跃度 |
| 项目审核 | ADMIN 手动审核 | MVP 阶段简单可靠 |
