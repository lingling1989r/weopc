### ❌ 发现的 API 问题

#### 严重问题 (1个)

**1. 项目过滤查询参数验证缺失**
- **端点**: `GET /api/v1/projects?revenueTier=TIER_1`
- **状态码**: 500 (应该是 400)
- **错误**: `Invalid value for argument 'where': Invalid value provided for field 'revenueTier'`
- **原因**: 查询参数使用了错误的枚举值，后端没有验证
- **正确值**: `TIER_0_1K`, `TIER_1K_5K`, `TIER_5K_10K`, `TIER_10K_30K`, `TIER_30K_50K`, `TIER_50K_100K`, `TIER_100K_PLUS`
- **影响**: 前端使用错误参数会导致 500 错误，用户体验差
- **修复建议**:
  ```typescript
  // 在路由中添加验证
  const validRevenueTiers = ['TIER_0_1K', 'TIER_1K_5K', ...];
  if (revenueTier && !validRevenueTiers.includes(revenueTier)) {
    throw new ValidationError('Invalid revenueTier value');
  }
  ```
